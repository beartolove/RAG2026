📋 CRUD 四個操作對應向量資料庫

向量資料庫的操作邏輯與一般關聯式資料庫相同，都支援 Create / Read / Update / Delete 四種基本操作，差別只在於存的是向量（Embedding 編碼後的結果）而非一般欄位資料。
C
CREATE
add() / upsert()
新增向量資料與對應的 Metadata 到資料庫
R
READ
query() / get()
依相似度搜尋 Top-K，或指定 ID 取出特定資料
U
UPDATE
upsert()
更新既有向量或 Metadata；若 ID 不存在則自動新增
D
DELETE
delete()
依 ID 或條件刪除指定的向量資料

🗄️ 本地端 vs 雲端：兩種向量資料庫選型

💻
ChromaDB
LOCAL — 本地端輕量資料庫

基於 SQLite 的本地端向量資料庫，安裝即用，資料存於本機路徑，無需額外基礎設施。

  • 適合 Prototype、本機開發
  • 自行指定本地端資料夾路徑
  • 無內建 Embedding，需自行提供 Embedding Function（如 OpenAI API）
  • 免費，零雲端成本
☁️
Pinecone
CLOUD — 雲端 SaaS 向量資料庫

基於 AWS / GCP / Azure 的雲端向量資料庫服務，API Key 認證後直接使用。

  • 適合上線封測、中小型生產環境
  • 內建 Embedding 模型，可直接傳文字查詢
  • Index = 你的資料庫，可管理多個
  • 付費服務，有免費額度

💻 ChromaDB 本地端 CRUD 實作

C — CREATE：建立資料庫並新增向量
Python — ChromaDB Create LOCAL
import chromadb
import numpy as np

# 建立本地端 ChromaDB（指定儲存路徑）
client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_or_create_collection("my_collection")

# 模擬一個 128 維向量（實際應由 Embedding 模型產生）
fake_vector = np.random.rand(128).tolist()

# 新增向量資料，附帶 ID 與 Metadata
collection.add(
    ids=["doc_001"],
    embeddings=[fake_vector],
    metadatas=[{"source": "numpy", "category": "test"}],
    documents=["這是一筆測試文件"]
)
R — READ：相似度查詢 + 指定 ID 取出
Python — ChromaDB Read LOCAL
# 相似度查詢：取出最像的前 3 筆
results = collection.query(
    query_embeddings=[fake_vector],
    n_results=3
)
# → 回傳按相似度排序的結果（ID、距離、Metadata、文件）

# 指定 ID 取出：精準查詢特定資料
item = collection.get(ids=["doc_001"])

# 取出第一筆（不指定 ID）
first = collection.get(limit=1)
U — UPDATE：使用 upsert 更新資料
Python — ChromaDB Update (Upsert) LOCAL
# Upsert = Update + Insert 合一
# 若 ID 已存在 → 更新；若不存在 → 新增
new_vector = np.random.rand(128).tolist()

collection.upsert(
    ids=["doc_001"],
    embeddings=[new_vector],
    metadatas=[{"source": "update_numpy"}],  # Metadata 同步更新
    documents=["更新後的文件內容"]
)
# 更新後重新 query → 結果會指向新的向量
INSERT
➕

ID 不存在時

自動新增一筆資料

+
UPDATE
✏️

ID 已存在時

自動更新現有資料（向量與 Metadata）

💡 Upsert = 現代向量資料庫普遍採用的方式，用一個操作取代傳統的「先查是否存在、再決定 Insert 或 Update」兩步流程

☁️ Pinecone 雲端 CRUD 實作

Python — Pinecone CRUD 完整範例 CLOUD
from pinecone import Pinecone
import numpy as np

# 初始化 Pinecone（API Key 認證）
pc = Pinecone(api_key="YOUR_API_KEY")

# 列出所有 Index（類似列出所有資料庫）
print(pc.list_indexes())

# 連接指定 Index
index = pc.Index("my-index")

# CREATE：新增向量（ID + 向量值 + Metadata）
vector = np.random.rand(128).tolist()
index.upsert(vectors=[{
    "id":     "1021",
    "values": vector,
    "metadata": {"source": "numpy"}
}])

# READ：用 ID 取出（values 預設遮蔽，需 fetch 展開）
result = index.fetch(ids=["1021"])
print(result.vectors["1021"].values)  # 展開後的向量值

# UPDATE：用 upsert 更新 Metadata
index.upsert(vectors=[{
    "id":     "1021",
    "values": np.random.rand(128).tolist(),
    "metadata": {"source": "update_numpy"}
}])

# 語意查詢：傳入向量，找最相似的前 3 筆
search_result = index.query(
    vector=vector,
    top_k=3,
    include_metadata=True
)
⚠ Pinecone 的 values 預設遮蔽：直接 get() 時向量值會被隱藏，需改用 fetch() 並展開才能看到原始向量數值。
Pinecone 搭配 Embedding：文字 → 向量 → 語意搜尋
Python — 文字 Embedding 後存入 Pinecone，再做語意查詢 CLOUD
from openai import OpenAI
client = OpenAI()

# 文字資料 → Embedding → 存入 Pinecone
documents = [
    "Apple is a technology company",
    "Google dominates search engines",
    "Apple is also a fruit",
    "Microsoft builds operating systems",
    "Amazon leads cloud computing",
]

for i, doc in enumerate(documents):
    vec = client.embeddings.create(input=doc, model="text-embedding-ada-002")
    index.upsert(vectors=[{"id": str(i), "values": vec.data[0].embedding}])

# 查詢：「跟 Apple 有關的有名公司是什麼？」
query_vec = client.embeddings.create(
    input="Which famous company is related to Apple?",
    model="text-embedding-ada-002"
).data[0].embedding

results = index.query(vector=query_vec, top_k=3)
# → 找出前 3 筆最相關（但若無 threshold，水果 Apple 也可能混入）

🔗 LangChain 封裝：更簡潔的操作方式

Python — LangChain 封裝 ChromaDB / Pinecone LANGCHAIN
from langchain_community.vectorstores import Chroma, Pinecone
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

# ── ChromaDB 版本 ──────────────────────────────
# Add Documents：自動 Embedding + 存入資料庫
vectorstore_chroma = Chroma.from_documents(
    documents=docs,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

# Similarity Search：自動 Embedding 查詢 + 相似度排序
results = vectorstore_chroma.similarity_search("Apple 相關公司", k=3)

# ── Pinecone 版本 ──────────────────────────────
vectorstore_pinecone = Pinecone.from_documents(
    documents=docs,
    embedding=embeddings,
    index_name="my-index"
)
results = vectorstore_pinecone.similarity_search("Apple 相關公司", k=3)

⚖️ 三種操作方式的選擇

🔧 原生 SDK（土炮）

直接使用 ChromaDB / Pinecone 的官方 SDK 操作，自行管理 Embedding 流程。

collection.add(...)
collection.query(...)
collection.upsert(...)
✓ 完全掌控，彈性最高
⚠ 需自己處理 Embedding 邏輯，程式碼較多
🔗 LangChain 封裝

透過 LangChain 統一介面，自動整合 Embedding 與資料庫操作。

from_documents(docs)
similarity_search(query)
add_documents(docs)
✓ 程式碼簡潔，兩行完成
⚠ 彈性稍低，需依賴 LangChain 版本
✅ 建議策略

初學 / 理解原理 → 用原生 SDK。快速開發 / 接入 RAG Pipeline → 用 LangChain 封裝。

理解底層 → 原生 SDK
快速接入 → LangChain
生產環境 → 視需求
✓ 兩者可互換，底層邏輯相同
⚠ LangChain 版本更新快，注意 API 變動

🏷️ Metadata 的重要性

每筆向量可附帶 Metadata 進行額外標記與篩選

Metadata 讓你在做語意搜尋的同時，可以疊加精準條件篩選（類似 SQL WHERE），在模糊搜尋的基礎上增加精確控制。

source: "pdf" category: "技術文件" author: "張維元" date: "2024-01" language: "zh-TW" chapter: 3
例如：只搜尋 source="pdf" 且 language="zh-TW" 的 Chunk，實現「語意模糊 + 條件精準」的混合查詢。

💡 核心觀念

向量資料庫的 CRUD 和一般資料庫概念完全相同，差異只在於存的是向量。理解 CRUD 操作是將 Embedding 文件整合進 RAG Pipeline 的基礎技能。
Upsert = Insert + Update 合一，是現代向量資料庫的主流操作方式。不需要先查再決定操作，一個 upsert 搞定所有寫入場景。
⚠ 沒有 Threshold 的語意搜尋會找到「最像但不相關」的結果。如查詢「Apple 相關公司」，第三筆可能返回「Apple 是一種水果」——搜尋結果必須搭配相似度閾值過濾，才能確保品質。
