Semantic Search 的
突破與局限

深入理解語意搜尋的核心原理：向量空間、Embedding 演進，以及為什麼它仍需要 RAG 來補足

張維元 · RAG 精準搜尋實戰 01–03
突破 vs 局限
🚀

Semantic Search 的突破

  • 不只看字面，能理解文字背後的語意
  • 解決同義詞問題（如：減脂 ≈ 飲食控制＋熱量限制）
  • 能理解使用者的查詢意圖（Query Intent）
  • 支援上下文語境理解與模糊搜尋
  • 透過向量距離衡量資料間的相似程度
⚠️

Semantic Search 的局限

  • 本質仍是「找文件」，停留在檢索層面
  • 沒有生成能力，無法產出整合後的答案
  • 沒有整合能力，無法彙整多份文件
  • 沒有推理能力，無法進行邏輯推導
  • 找到的結果不夠友善，缺乏 User-Friendly 的呈現
語意搜尋的核心概念

Semantic Search = RAG 的前半段

語意搜尋在做的事，其實就是 RAG 的前半段流程：把原始文字透過 Embedding 模型（編碼模型）打進一個向量空間，再透過相似度比對找到語意相近的文本。

在這個向量空間當中，距離近 = 語意相似，距離遠則代表語意不同。電腦能在高維空間中計算距離，這就是語意搜尋能突破關鍵字比對的根本原因。

⚠️ 關鍵提醒：Embedding 做得好不好，直接決定向量空間的品質，進而影響整個搜尋結果的準確性。Embedding 是 Semantic Search 的關鍵，也是 RAG 系統的地基。

NLP 如何讓電腦讀懂語言
Step 01
✂️

斷詞 Tokenization

將句子切成最小文本單位。中文無空格，切法影響語意（「才能」vs「才＋能」）

Step 02
🔢

向量化 Vectorization

將每個詞轉換為數值向量，讓電腦可以進行數學運算

Step 03
🧩

語意理解 Semantics

分析詞語之間的上下文關係，保留語意資訊在向量空間中

Step 04
🎯

相似度計算 Similarity

計算向量空間中各點距離，距離近的視為語意相似

Embedding 技術演進
📋

One-Hot Encoding（最早期）

每個詞用一個超長的稀疏向量表示，詞表有多大向量就有多長，只標記該詞的位置為 1，其餘為 0。

✓ 概念簡單
✗ 維度爆炸、無順序、完全沒有語意關係
🧮

Bag of Words (BoW)＋詞頻統計

計算每個詞在文件中出現的次數（詞頻），不考慮詞語順序與語法，只看「頻率」。

✓ 簡單可計算
✗ 沒有順序資訊、完全忽略語意
⚖️

TF-IDF（詞頻加權）

在 BoW 基礎上加入「逆文件頻率」修正：降低常見詞（如「的、是」）的權重，突出各文件中真正重要的關鍵字。
公式：TF（詞在本文的頻率） × IDF（在全部文件中的稀有程度）

✓ 過濾雜訊詞，提取關鍵字效果佳
✗ 仍無語意，詞語間無關聯性
🧠

Word2Vec（Google 提出，語意突破）

利用神經網路學習每個詞的「上下文關係」，將詞語壓縮為密集的低維向量。相似的字在向量空間中距離近，不相似的距離遠——這才是真正具有語意的向量空間。

✓ 向量具有語意、支援向量運算（如：國王 − 男人 + 女人 ≈ 女王）
✗ 每個詞只有一個固定向量，無法表示一詞多義
向量空間直覺理解

距離 = 語意相似程度

在向量空間中，每個詞或句子都對應一個點。Embedding 的目標是讓語意相近的詞「聚在一起」，語意遙遠的詞「相距較遠」。

搜尋時，輸入的查詢會被轉換為一個向量，系統找出向量空間中距離最近的點作為搜尋結果——這就是語意模糊搜尋的本質。

稀疏矩陣（BoW） → 壓縮 → 密集語意向量（Word2Vec / Embedding）

與 RAG 的關係
🔗

Semantic Search 解決了「找到語意相關的文件」這個問題，但找到了之後呢？結果仍然是原始文字、不夠易讀、也無法推理整合。RAG 正是在 Semantic Search 的基礎上，加入 LLM 的生成能力，讓「找到的資料」能夠被理解、被整合、被轉化為友善且有意義的回答。

本集核心重點

🎯 五大帶走重點

1

Semantic Search 是搜尋革命，但不是終點——它解決了關鍵字比對的瓶頸，卻仍缺乏生成、整合與推理能力。

2

Embedding 是一切的地基——向量空間的品質決定語意搜尋的好壞，也是 RAG 系統能否準確運作的根本。

3

NLP 的本質：文字 → 向量——電腦不懂語言只懂數字，NLP 做的就是把人類語言轉成可計算的向量，讓電腦保留語意去做運算。

4

Embedding 演進：BoW → TF-IDF → Word2Vec——從只看頻率到真正理解語意關係，Word2Vec 是語意向量的重要里程碑。

5

語意搜尋 = RAG 的前半段——理解 Semantic Search 的原理，就是在理解 RAG 的檢索核心；後半段則由 LLM 負責生成與整合。
