準備語料庫（Corpus）

準備數筆文件作為知識來源。範例使用 RAG 相關說明文字，每一句話代表一個知識片段（Chunk）。

Python — 定義語料庫
# 模擬 RAG 知識庫（4 筆資料）
corpus = [
    "RAG 是檢索增強生成技術",
    "RAG 結合了搜尋與語言模型",
    "如何實作 RAG",
    "嵌入和向量資料庫是 RAG 的基礎",
]
🔢

TF-IDF 向量化語料庫

使用 sklearn 的 TfidfVectorizer 將文字轉換為稀疏向量矩陣（4×6），每個字作為一個維度，數值代表在該句中的相對重要性。

Python — 建立 TF-IDF 向量空間
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

vectorizer = TfidfVectorizer()
corpus_vectors = vectorizer.fit_transform(corpus)
# 結果：4×6 稀疏矩陣（4 句話，6 個關鍵詞維度）
🔍

查詢向量化 → Cosine 相似度比對 → 取 Top-K

將使用者的提問同樣轉換為 TF-IDF 向量，再與語料庫中每一筆向量計算 Cosine 相似度，取出最相似的 K 筆資料作為上下文。

Python — 語意檢索函式
def retrieve(query, k=2, threshold=0.0):
    query_vec = vectorizer.transform([query])
    scores = cosine_similarity(query_vec, corpus_vectors)[0]

    # 依相似度排序，取 Top-K
    ranked = scores.argsort()[::-1][:k]

    # 套用 threshold 門檻過濾
    results = [corpus[i] for i in ranked if scores[i] >= threshold]
    return results
📎

將檢索結果注入 Prompt → LLM 生成答案

把 Top-K 的結果組合成「上下文參考資料」，拼入 Prompt，讓 LLM 根據這些資料回答，而非憑自身訓練記憶回答。

Python — 上下文注入與 LLM 生成
from openai import OpenAI
client = OpenAI()

def rag_answer(question, threshold=0.3):
    context_docs = retrieve(question, k=2, threshold=threshold)

    if not context_docs:
        return "找不到相關資料，無法回答。"

    context = "\n".join(context_docs)
    prompt = f"""根據以下上下文回答問題：
上下文：{context}
問題：{question}
請根據上下文作答。"""

    res = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}]
    )
    return res.choices[0].message.content

🔢 TF-IDF 向量矩陣長什麼樣

4 筆語料轉換後得到 4×6 的稀疏矩陣：每列是一句話，每欄是一個關鍵詞。數值代表該詞在這句話中的 TF-IDF 權重。出現在所有句子中的詞（如「RAG」）權重會被壓低，句子獨有的詞權重反而更高。

⚖️ 有上下文 vs 無上下文的差異

✅ WITH CONTEXT（RAG 模式）

先從語料庫找到相似段落，注入 Prompt，LLM 依據這些資料生成答案，回答內容受限於你的知識庫，不會憑空捏造。

🎚️ Threshold 相似度門檻的權衡

即使查詢內容不在資料庫中（如「請問 Agent 是什麼」），RAG 仍會勉強找出「最像」的結果並生成奇怪答案。加上 Threshold 門檻，可過濾掉相似度不足的結果，避免無中生有。

💡 從這次實作學到的核心觀念

RAG 的本質就三步：把資料放到向量空間 → 用相似度找出最像的片段 → 讓 LLM 放大並生成回答。框架只是讓這三步更方便，核心邏輯不變。
TF-IDF vs Embedding 的差異：TF-IDF 是稀疏的詞頻統計向量，Embedding 是稠密的語意向量。TF-IDF 適合快速原型驗證，正式 RAG 應使用 Embedding 模型以獲得更好的語意理解能力。
Threshold 的設定沒有標準答案，需根據實際資料和業務容忍度反覆實驗：太鬆有幻覺風險，太緊會拒絕太多正當問題。
