二階檢索與 Re-Rank 排序機制

張維元 ── RAG 精準搜尋實戰 03-05 | 抓大放小策略 × CrossEncoder Re-Rank × Hybrid Search × Multi-Query × RAGAS 評估

🎯 核心策略：抓大放小

向量搜尋找到的 K 句話「很像」，但不代表「最正確」——相似度不等於相關性。
解法：先用寬鬆的條件多撈一點（高 Recall），再透過精準的 Re-Rank 排序機制篩選出真正重要的（高 Precision）。

⚡ 兩階段運作機制

01
STAGE 1 — 寬鬆召回

向量相似度初篩

用語意向量快速撈出候選集，寧可多抓也不要漏掉正確答案。追求 Recall 高，不強求 Precision。

速度快 成本低 Top-20 ～ Top-50 可能有雜訊
→
02
STAGE 2 — 精準排序

CrossEncoder Re-Rank

對候選集進行精細的二次排序，判斷每筆 Chunk 與 Query 的真實相關性，追求 Precision 高。

準確度高 最終 Top-3 ～ 5 速度較慢 成本較高

🎬 實作範例：電影資料庫搜尋

實作查詢：「我想看蝙蝠的超級英雄動作片」
關鍵字拆解：蝙蝠 + 超級英雄 + 動作片
STEP 1 — 向量搜尋初篩（Sentence Transformers + ChromaDB）
Python — 建立向量資料庫並初篩 STAGE 1
from sentence_transformers import SentenceTransformer
import chromadb

# 初始化 Embedding 模型
model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")

# 電影資料（title 存 metadata，content 存 document）
movies = [
    {"title": "黑暗騎士",       "content": "蝙蝠俠對抗小丑的暗黑英雄動作片"},
    {"title": "蝙蝠俠大戰超人", "content": "DC 兩大英雄的史詩對決"},
    {"title": "蜘蛛人：無家日", "content": "多元宇宙蜘蛛人的奇幻冒險"},
    {"title": "神偷奶爸",       "content": "壞蛋變成愛心奶爸的喜劇動畫"},
]

# 建立向量資料庫
client     = chromadb.Client()
collection = client.create_collection("movies")
for i, m in enumerate(movies):
    vec = model.encode(m["content"]).tolist()
    collection.add(ids=[str(i)], embeddings=[vec],
                    metadatas=[{"title": m["title"]}],
                    documents=[m["content"]])

# 查詢：取前 4 筆候選（寬鬆召回）
query    = "我想看蝙蝠的超級英雄動作片"
q_vec    = model.encode(query).tolist()
results  = collection.query(query_embeddings=[q_vec], n_results=4)
初篩排名電影名稱向量相似度分數備註
1黑暗騎士0.82語意最接近蝙蝠英雄
2蝙蝠俠大戰超人0.79DC 英雄動作片
3神偷奶爸0.51動作？語意模糊混入
4蜘蛛人：無家日0.48超級英雄語意相關
⚠ 初篩問題：「神偷奶爸」透過語意模糊混入前4，這就是為什麼需要 Re-Rank——初篩只保證 Recall，不保證 Precision。
STEP 2 — CrossEncoder Re-Rank 精準排序
Python — CrossEncoder Re-Rank 實作 STAGE 2
from sentence_transformers import CrossEncoder

# 使用支援中文的 CrossEncoder 模型（BGE Re-Ranker）
reranker = CrossEncoder("BAAI/bge-reranker-base")

# 取出初篩結果
candidates = results["documents"][0]     # 文件內容
titles     = [m["title"] for m in results["metadatas"][0]]

# 用「標題 + 內文」拼合後做 Re-Rank（效果最佳）
pairs  = [(query, t + " " + c) for t, c in zip(titles, candidates)]
scores = reranker.predict(pairs)

# 依分數降序排列
ranked = sorted(zip(titles, candidates, scores),
                key=lambda x: x[2], reverse=True)

for title, content, score in ranked:
    print(f"{title}: {score:.4f}")

🤖 Re-Rank 模型選擇：英文 vs 中文

❌ 英文模型用於中文

使用預設的英文 CrossEncoder 模型排序中文內容，語意對齊效果不佳，排序結果不符合預期。

例如「蜘蛛人：無家日」可能因英文詞根匹配而排名虛高，蓋過更相關的蝙蝠俠系列。

cross-encoder/ms-marco-MiniLM-L-6-v2
✗ 對中文排序結果不可信
✅ 中文專用模型（BGE）

使用 BAAI 出品的 BGE（BAAI General Embedding）Re-Ranker，專為中英文語料優化，排序品質顯著提升。

排序後「黑暗騎士」正確拉升至首位，蝙蝠俠大戰超人依序排列，符合語意預期。

BAAI/bge-reranker-base
✓ 中文環境推薦，HuggingFace 可取得

標題排序

🔀 三種進階 Retrieval 策略實作

1
單純二階檢索（Vector → Re-Rank）
基礎做法，適合大多數場景

用向量搜尋取出 Top-K 候選，再用 CrossEncoder 精排，最後取前 N 筆送入 LLM 生成回答。

向量查詢 Top-K → CrossEncoder Re-Rank → LLM 生成
Python — 二階檢索 + LLM 生成 BASIC
# 取出 Re-Rank 後的前 3 筆作為上下文
top_docs = [content for _, content, _ in ranked[:3]]
context  = "\n".join(top_docs)

prompt = f"""根據以下電影資料回答：
{context}

問題：{query}
請根據上述資料推薦最符合的電影。"""

# 送入 LLM 生成答案
response = openai_client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}]
)
2
Hybrid Search（向量 + BM25 → Re-Rank）
兼顧語意與精準關鍵字，適合複雜查詢

兩種搜尋分別取回候選集，合併去重後統一送入 Re-Ranker 精排，同時保有語意模糊 + 精準關鍵字的能力。

向量搜尋 + BM25 關鍵字 → 合併去重 → Re-Rank → LLM
Python — Hybrid Search 合併候選集 HYBRID
from rank_bm25 import BM25Okapi

# BM25 關鍵字搜尋
keywords   = ["英雄", "動作"]
corpus     = [m["content"].split() for m in movies]
bm25       = BM25Okapi(corpus)
bm25_scores= bm25.get_scores(keywords)
bm25_top   = bm25_scores.argsort()[::-1][:4]

# 合併：向量搜尋結果 + BM25 結果，去重
vector_ids = set(results["ids"][0])
bm25_ids   = set(str(i) for i in bm25_top)
merged_ids = vector_ids | bm25_ids   # 取聯集

# 對合併後的候選集統一 Re-Rank
merged_docs = [movies[int(i)] for i in merged_ids]
# ... 接 CrossEncoder Re-Rank 流程
3
Multi-Query（查詢擴展 → 多路搜尋 → Re-Rank）
用 LLM 改寫查詢，應對模糊或口語化的問題

使用者說「高譚市的守護者」，資料庫裡可能沒有這個詞，但 LLM 能將其擴展為「蝙蝠俠、蝙蝠、暗黑騎士」等查詢，提升召回率。

原始 Query → LLM 改寫擴展 → 多路向量查詢 → 去重合併 → Re-Rank
Python — Multi-Query 查詢擴展 MULTI-QUERY
# 用 LLM 把模糊查詢擴展成多個關鍵字
expand_prompt = f"""把「{query}」改寫成 3 個不同的搜尋查詢。
每行一個，只輸出查詢詞，不要其他說明。"""

expanded_queries = llm.generate(expand_prompt).split("\n")
# 例如：["蝙蝠俠動作片", "DC 超級英雄電影", "暗黑英雄動作"]

# 對每個擴展查詢分別做向量搜尋
all_candidates = []
for q in expanded_queries:
    q_vec  = model.encode(q).tolist()
    result = collection.query(query_embeddings=[q_vec], n_results=3)
    all_candidates.extend(result["documents"][0])

# 去重後統一 Re-Rank
unique_candidates = list(dict.fromkeys(all_candidates))
# ... 接 CrossEncoder Re-Rank 流程

📊 RAGAS 評估框架：量化 RAG 效果

做完 RAG 系統後，怎麼知道效果好不好？用 RAGAS（RAG Assessment）框架，以四個指標量化評估，找出系統瓶頸。
✅
FAITHFULNESS — 正確性

生成答案有沒有基於參考資料

LLM 的回答是否忠實於檢索到的 Context，還是自己「發揮」了超出資料的內容（幻覺）。

「答案是從 Context 生成的嗎？」
🎯
ANSWER RELEVANCY — 答案相關性

生成答案有沒有真正回答問題

回答的內容是否切合使用者的問題，避免答非所問或提供過多不相關資訊。

「答案有回答到問題嗎？」
🔍
CONTEXT PRECISION — 上下文精準率

取回的 Context 有多少是真正有用的

衡量檢索到的 Chunk 中，有多少比例與回答問題真正相關——去除雜訊的能力。

「找到的 Context 夠精準嗎？」
📡
CONTEXT RECALL — 上下文召回率

應被找到的資料有沒有被找到

衡量所有回答問題所需的資訊是否都被成功檢索出來——不漏掉重要資料的能力。

「需要的 Context 都找齊了嗎？」
Python — RAGAS 評估使用範例 RAGAS
from ragas import evaluate
from ragas.metrics import (
    faithfulness, answer_relevancy,
    context_precision, context_recall
)
from datasets import Dataset

# 準備評估資料集（Ground Truth 問題 + 答案）
eval_data = {
    "question":  ["推薦一部蝙蝠俠相關的動作片"],
    "answer":    [llm_generated_answer],         # LLM 生成的答案
    "contexts":  [[top_docs[0], top_docs[1]]],  # 取到的 Context
    "ground_truth": ["黑暗騎士是蝙蝠俠系列中評價最高的動作片"]
}

dataset = Dataset.from_dict(eval_data)
result  = evaluate(
    dataset,
    metrics=[faithfulness, answer_relevancy,
             context_precision, context_recall]
)
print(result)
# → {'faithfulness': 0.92, 'answer_relevancy': 0.88,
#    'context_precision': 0.75, 'context_recall': 0.83}

💡 核心觀念

「很像」不等於「最對」——Re-Rank 正是用來補這個差距。向量相似度衡量的是方向接近，CrossEncoder Re-Rank 衡量的是語意相關性，兩者目標不同，需要搭配使用。
Re-Rank 時，輸入「標題 + 內文」效果優於單一欄位。讓模型同時看到識別標籤（標題）與語意內容（內文），排序結果更符合預期。
Multi-Query 解決「使用者不會精確表達」的問題。用 LLM 把模糊查詢改寫成多個具體查詢，擴大召回範圍，再用 Re-Rank 精選最終結果。
⚠ Re-Rank 模型必須選對語言。英文模型用於中文排序，效果不可信。中文環境推薦使用 BAAI/BGE 系列模型，可在 HuggingFace 取得。

📋 三種 Retrieval 策略與 RAGAS 評估對照

策略流程優點適合場景
Vector → Re-Rank 向量初篩 → CrossEncoder 精排 實作簡單，效果明顯提升 一般問答、推薦系統
Hybrid → Re-Rank 向量 + BM25 合併 → Re-Rank 同時兼顧語意與精準關鍵字 含專有名詞、法規條文查詢
Multi-Query → Re-Rank LLM 改寫 → 多路查詢 → Re-Rank 應對模糊口語化查詢，召回率高 使用者輸入不精確的場景
RAGAS 評估 四指標：正確性/相關性/精準率/召回率 量化 RAG 品質，找出瓶頸層 所有 RAG 系統上線前的評估
