📏 評估指標：Precision vs Recall

📗 RECALL — 召回率

答案有沒有被「找到」

Top-K 取回的 Chunk 中，有沒有包含標準答案的關鍵字。衡量的是「漏網率」——正確資訊是否進入了 LLM 的上下文。

命中關鍵字數 ÷ 所有應命中關鍵字數
⚠ Chunk 越大，Recall 越高，但不代表越好（雜訊也同時增加）
🔵 PRECISION — 精準率

找到的結果有多「乾淨」

取回的 Chunk 中，真正相關的比例有多高。衡量的是「雜訊率」——有多少 Chunk 是無用的干擾。

相關 Chunk 數 ÷ Top-K 取回的 Chunk 總數
⚠ Chunk 越小，Precision 越高，但可能漏掉完整答案

💻 核心實作程式碼

STEP 1 — 設定文本、問題與 Ground Truth
Python
# 測試文本（知識來源）
text = "珍珠奶茶起源於 1980 年代的台灣台南..."

# Ground Truth：問題 + 標準答案關鍵字
test_cases = [
    {"question": "珍珠奶茶起源於哪個年代？",
     "answer_keywords": ["1980", "台灣"]},
    {"question": "珍珠奶茶最早在哪個城市出現？",
     "answer_keywords": ["台南", "台中"]},
]
STEP 2 — 三組不同參數切割
Python
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 組 A：小 Chunk，無 Overlap
splitter_A = RecursiveCharacterTextSplitter(chunk_size=50,  chunk_overlap=0)

# 組 B：大 Chunk，無 Overlap
splitter_B = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=0)

# 組 C：中 Chunk，有 Overlap（推薦起始值）
splitter_C = RecursiveCharacterTextSplitter(chunk_size=150, chunk_overlap=20)

chunks_A = splitter_A.split_text(text)
chunks_B = splitter_B.split_text(text)
chunks_C = splitter_C.split_text(text)
STEP 3 — Embedding + Cosine Similarity 取 Top-K
Python
from sklearn.metrics.pairwise import cosine_similarity
from openai import OpenAI
import numpy as np

client = OpenAI()

def get_embedding(text):
    res = client.embeddings.create(input=text, model="text-embedding-ada-002")
    return res.data[0].embedding

def retrieve_top_k(query, chunks, k=3):
    query_vec  = np.array(get_embedding(query)).reshape(1, -1)
    chunk_vecs = np.array([get_embedding(c) for c in chunks])
    scores     = cosine_similarity(query_vec, chunk_vecs)[0]
    top_k_idx  = scores.argsort()[::-1][:k]
    return [chunks[i] for i in top_k_idx]
STEP 4 — 計算 Precision / Recall 並比較各組
Python
def evaluate(test_cases, chunks, k=3):
    total_recall = total_precision = 0

    for case in test_cases:
        retrieved      = retrieve_top_k(case["question"], chunks, k)
        retrieved_text = " ".join(retrieved)
        keywords       = case["answer_keywords"]

        # Recall：答案關鍵字有沒有出現在 Top-K 中
        recall_hits = sum(1 for kw in keywords if kw in retrieved_text)
        recall      = recall_hits / len(keywords)

        # Precision：Top-K 中有多少 Chunk 包含關鍵字
        prec_hits = sum(
            1 for chunk in retrieved
            if any(kw in chunk for kw in keywords)
        )
        precision = prec_hits / len(retrieved)

        total_recall    += recall
        total_precision += precision

    n = len(test_cases)
    return {"recall": total_recall/n, "precision": total_precision/n}

# 比較三組
for name, chunks in [("A (size=50)", chunks_A),
                       ("B (size=500)", chunks_B),
                       ("C (size=150+ov)", chunks_C)]:
    scores = evaluate(test_cases, chunks)
    print(f"組別 {name} → Recall: {scores['recall']:.2f} | Precision: {scores['precision']:.2f}")

🔁 建議的實驗迭代流程

01

準備 Ground Truth 資料集

針對你的知識來源，準備 10–20 筆「問題 + 答案關鍵字」，這是整個評估的基準，越貼近真實使用者問題越好。

02

設計 3–5 組 Chunk 參數組合

從小到大設定不同的 Size（如 50 / 150 / 300 / 500），搭配有 / 無 Overlap，確保實驗組之間差異足夠明顯。

03

對每組計算 Precision / Recall

執行 Embedding → Cosine → Top-K 流程，對照 Ground Truth 計算每組的評估分數，記錄結果。

04

選擇綜合最佳組合作為基準

選出 Precision 與 Recall 綜合表現最佳的參數組合，作為下一步優化的起點。

05

切換 Embedding 模型，重複實驗

鎖定最佳 Chunk 參數後，再嘗試不同的 Embedding 模型，觀察向量品質對分數的影響，找到整體最佳組合。

06

進入 Retrieval 層調教

確定 Chunk 與 Embedding 後，進一步優化 Top-K 數量、Retrieval 策略與 Re-Ranking，持續提升整體 RAG 品質。

💡 核心觀念

沒有評估等於盲測。不跑實驗直接上線，你對 RAG 的實際品質一無所知。Ground Truth 是最小成本的品質防線，只需「問題 + 答案關鍵字」就能建立。
Chunk 越小不代表越好。Precision 高只代表雜訊少，但 Recall 低意味著正確答案根本沒被找到——LLM 再厲害，上下文裡沒有答案，它也無從生成。
最終目標只有一個：找到「剛剛好」的 Chunk——它能包含完整的答案、又不因為太長而讓向量語意失真。這個點因資料而異，只有實驗才能告訴你答案。
