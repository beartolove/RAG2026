🪜 實作四步驟

01

安裝環境：PostgreSQL + pgvector

在 Colab 中使用 ! 驚嘆號在背景執行系統指令，安裝 PostgreSQL 並手動編譯安裝 pgvector 擴充套件，使資料庫支援向量型別。

shell — Colab 環境安裝
# 安裝 PostgreSQL 並啟動服務
!apt-get install -y postgresql postgresql-contrib
!service postgresql start

# 手動編譯安裝 pgvector 擴充套件
!git clone https://github.com/pgvector/pgvector.git
!cd pgvector && make && make install

💡 若在本地端執行，只需確保已安裝支援 pgvector 的 PostgreSQL 版本，不需上述複雜步驟。

02

建立資料表：設計含 Embedding 欄位的 Table

連接 PostgreSQL，建立 hotel_reviews 資料表，其中向量欄位長度設為 1536，對應 OpenAI Embedding Model 的輸出維度。

Python — 建立 hotel_reviews 資料表
import psycopg2

conn = psycopg2.connect(
    dbname="my_vector_table",
    user="postgres", password="your_password"
)
cursor = conn.cursor()

# 啟用 pgvector 擴充
cursor.execute("CREATE EXTENSION IF NOT EXISTS vector")

# 建立含 1536 維向量欄位的資料表
cursor.execute("""
CREATE TABLE hotel_reviews (
    id       SERIAL PRIMARY KEY,
    name     TEXT,
    category TEXT,
    review   TEXT,
    vector   VECTOR(1536)   -- OpenAI Embedding 維度
)
""")
conn.commit()

向量長度 1536 來自 OpenAI text-embedding-ada-002 模型的輸出規格。

03

寫入資料：文字 → Embedding → 存入資料庫

使用 OpenAI API 將每筆評論文字轉換為向量，連同原始欄位一起寫入資料庫。進出資料庫皆需進行編碼（向量化）與解碼。

Python — 向量化並寫入資料
from openai import OpenAI
client = OpenAI(api_key="YOUR_API_KEY")

def get_embedding(text):
    res = client.embeddings.create(
        input=text,
        model="text-embedding-ada-002"
    )
    return res.data[0].embedding

# 範例資料
reviews = [
    {"name": "海景別墅", "category": "resort",   "review": "這家別墅有游泳池"},
    {"name": "城市商旅", "category": "business", "review": "市中心的商務城市飯店"},
]

for r in reviews:
    vec = get_embedding(r["review"])   # 文字 → 向量
    cursor.execute(
        "INSERT INTO hotel_reviews (name, category, review, vector) VALUES (%s,%s,%s,%s)",
        (r["name"], r["category"], r["review"], vec)
    )
conn.commit()
04

語意查詢：向量距離比對 + 條件篩選

將查詢語句同樣向量化，利用 pgvector 內建的距離運算子（<->）找出最相近的資料，支援混合使用語意模糊比對與欄位精準篩選。

Python — 語意查詢函式
def semantic_search(query_text, category=None, top_k=3):
    query_vec = get_embedding(query_text)   # 查詢文字 → 向量

    if category:
        # 先用 category 精準篩選，再做向量距離排序
        cursor.execute("""
            SELECT name, review, vector <-> %s AS distance
            FROM hotel_reviews
            WHERE category = %s
            ORDER BY distance LIMIT %s
        """, (query_vec, category, top_k))
    else:
        cursor.execute("""
            SELECT name, review, vector <-> %s AS distance
            FROM hotel_reviews
            ORDER BY distance LIMIT %s
        """, (query_vec, top_k))

    return cursor.fetchall()

# 範例查詢
semantic_search("有推薦的商務飯店嗎")           # 純語意查詢
semantic_search("私人游泳池", category="resort") # 語意 + 欄位篩選

🧠 關鍵技術概念

📐

向量維度 1536

OpenAI text-embedding-ada-002 輸出的向量長度，存入 pgvector 欄位時需對應宣告。

📏

向量距離運算

pgvector 內建 <-> 歐氏距離運算子，不須自行實作相似度計算，直接於 SQL 中使用。

🔀

混合查詢策略

可先用傳統 WHERE 條件精準篩選，再對結果集做向量相似度排序，兼顧精準與模糊。

🔄

編碼 / 解碼對稱

資料寫入前與查詢時，皆需經過 Embedding 轉換，確保向量空間的對應一致性。
